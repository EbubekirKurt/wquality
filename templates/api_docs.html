API Documentation

Integrate Weather Quality predictions into your applications

Base URL

All API endpoints are available at:

https://huggingface.co/spaces/ebubekirkurtt/weather-quality-app/api

Get Forecast

GET /api/forecast?city={city_name}&history={true|false}

Get weather quality forecast for a specific city.

Parameters

Parameter Type Required Description
city string Yes City name (e.g., "Istanbul", "New York")
history boolean No Include historical data (default: false)

Response

{ "city": "Istanbul", "forecast": { "forecasts": [ { "day": "Today", "temp": 22.5, "humidity": 65.0, "cloud_cover": 2.5, "quality": 78.3, "icon": "fa-sun", "category": "Good", "color": "#0984e3", "explanation": "Excellent conditions! Ideal Temperature contributing to a high score.", "lat": 41.0082, "lon": 28.9784 } ], "activities": { "Running": {"icon": "fa-person-running", "day": "Tomorrow", "score": 82}, "Picnic": {"icon": "fa-utensils", "day": "Saturday", "score": 88} }, "history": [] } }

Get Other Cities

GET /api/other-cities

Get today's weather quality for default cities (cached for 10 minutes).

Response

{ "New York": { "temp": 18.5, "humidity": 55.0, "quality": 75.2, "icon": "fa-cloud-sun", "category": "Good", "color": "#0984e3", "lat": 40.7128, "lon": -74.0060 }, "Tokyo": { "temp": 15.3, "humidity": 60.0, "quality": 82.1, ... } }

Search City

GET /api/search-city?q={query}

Search for cities by name (autocomplete).

Parameters

Parameter Type Required Description
q string Yes Search query (minimum 3 characters)

Response

{ "results": [ { "name": "Istanbul", "display": "Istanbul, Istanbul, Turkey", "lat": 41.0082, "lon": 28.9784 } ] }

Get Historical Data

GET /api/history?city={city_name}

Get historical temperature trends for the last 5 years.

Response

{ "history": [ {"year": 2020, "temp": 18.5}, {"year": 2021, "temp": 19.2}, {"year": 2022, "temp": 19.8}, {"year": 2023, "temp": 20.1}, {"year": 2024, "temp": 20.5} ] }

Forecast by Coordinates

GET /api/forecast-by-coords?lat={latitude}&lon={longitude}

Get forecast using geographic coordinates.

Caching

All endpoints support HTTP caching:

  • ETag: Conditional requests supported
  • Cache-Control: Public caching with appropriate max-age
  • 304 Not Modified: Returned when content unchanged

Cache duration: 5-10 minutes depending on endpoint.

Error Handling

All endpoints return standard HTTP status codes:

  • 200: Success
  • 304: Not Modified (cached)
  • 400: Bad Request (missing parameters)
  • 404: Not Found (city not found)
  • 500: Server Error

Error responses include a message:

{ "error": "City 'InvalidCity' not found." }
